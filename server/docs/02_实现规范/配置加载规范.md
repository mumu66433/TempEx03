# 配置加载规范

## 当前机制

- Excel 原始文件统一放在 `server/src/assets/config`。
- 服务启动时由 `server/src/server.js` 调用 `loadExcelConfigs()` 预加载全部工作簿。
- 读取逻辑集中在 `server/src/config/excelLoader.js`。
- 业务层通过 `getExcelWorkbook()` 或 `getExcelSheet()` 读取内存缓存，不直接反复读文件。

## Excel 解析约定

- 仅加载 `.xlsx` 文件。
- 忽略以 `~$` 开头的临时文件和隐藏文件。
- 每个工作表默认使用前三行作为元数据：
  - 第 1 行：字段名
  - 第 2 行：字段说明
  - 第 3 行：字段类型
- 第 4 行起为正式数据。

## 已支持字段类型

| 类型 | 说明 |
| --- | --- |
| `int` / `intid` | 必须解析为整数 |
| `number` | 解析为数值 |
| `string` | 转成字符串 |
| `json` | 按 JSON 解析 |
| `array` | 支持 JSON 数组或逗号分隔字符串 |
| `any` | 保持原值 |

## 开发约束

- 新增配置表时，优先沿用当前三行表头格式，避免为单个表写特例。
- 业务 service 不应直接假设 Excel 原始单元格类型，必须处理空值和类型转换结果。
- 若需要支持新的字段类型，应先补充本规范，再修改 `excelLoader.js`。
- 若配置表加载失败，错误信息应尽量包含工作簿名、工作表名、字段名和行号，方便策划与程序共同排查。

## 后续建议

- 当前加载结果只保存在内存中，适合开发和轻量模板阶段。
- 如果未来配置量增长明显，可以再评估热更新、版本缓存或构建期预处理方案。
