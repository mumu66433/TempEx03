# 账号接口

本文档覆盖当前账号注册与登录接口，对应代码位于 `server/src/routes/authRoutes.js` 和 `server/src/services/authService.js`。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/api/auth/register` | 注册账号；未传账号时自动生成 |
| `POST` | `/api/auth/login` | 使用账号密码登录 |

## POST /api/auth/register

### 用途

- 创建新的 `Player` 账号。
- 当前默认密码为 `123456`；如果调用方显式传入密码，则按传入值注册。
- 当 `account` 为空时，服务端会自动生成以 `P` 开头的唯一账号。

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 否 | 玩家账号，服务端会转成大写并去除首尾空格 |
| `password` | `string` | 否 | 玩家密码，缺省时使用默认密码 `123456` |

### 成功响应

HTTP 状态码：`201`

```json
{
  "ok": true,
  "player": {
    "id": "ckxxxx",
    "account": "PABC1234",
    "nickname": "PABC1234",
    "lastLoginAt": null,
    "createdAt": "2026-04-25T00:00:00.000Z",
    "updatedAt": "2026-04-25T00:00:00.000Z"
  },
  "defaultPassword": "123456"
}
```

### 失败响应

HTTP 状态码：`400`

```json
{
  "ok": false,
  "error": "账号已存在"
}
```

### 说明

- 返回体中的 `defaultPassword` 实际上表示本次注册使用的密码，当前命名沿用现有实现。
- `nickname` 当前默认与账号一致，后续如需开放昵称自定义，应先更新接口文档和服务逻辑。

## POST /api/auth/login

### 用途

- 校验账号密码并返回玩家基础信息。
- 登录成功后更新 `lastLoginAt`。

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号，服务端会转成大写并去除首尾空格 |
| `password` | `string` | 是 | 登录密码 |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "player": {
    "id": "ckxxxx",
    "account": "PABC1234",
    "nickname": "PABC1234",
    "lastLoginAt": "2026-04-25T00:10:00.000Z",
    "createdAt": "2026-04-25T00:00:00.000Z",
    "updatedAt": "2026-04-25T00:10:00.000Z"
  }
}
```

### 失败响应

HTTP 状态码：`401`

```json
{
  "ok": false,
  "error": "密码错误"
}
```

### 说明

- 当前未引入 token、session 或刷新机制，接口只负责最基础的账号校验。
- 若后续增加鉴权体系，应新增独立专题文档，不直接覆盖本文结构。
