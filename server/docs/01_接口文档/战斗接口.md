# 战斗接口

本文档覆盖当前为 V0 前端骨架补齐的最小战斗会话接口。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/battle/session` | 读取当前账号的战斗会话 |
| `POST` | `/api/battle/session/start` | 为当前账号创建或重置最小战斗会话 |
| `POST` | `/api/battle/session/settle` | 结算当前战斗会话，并回写章节进度 |

## GET /api/battle/session

### 请求参数

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | query | 是 | 玩家账号 |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "session": {
    "id": "cm123",
    "chapterId": 1,
    "layerIndex": 1,
    "waveIndex": 1,
    "missionId": 1,
    "waveType": "普通波",
    "enemyId": 1,
    "enemyName": "鸡",
    "enemyCount": 1,
    "enemyLife": 3,
    "enemyAtk": 2,
    "status": "active",
    "result": null,
    "chapter": {
      "id": 1,
      "name": "银杏村"
    }
  }
}
```

## POST /api/battle/session/start

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `chapterId` | `number` | 否 | 目标章节；为空时使用玩家当前章节 |

### 说明

- 会根据当前章节的首个 `Mission` 配置生成最小战斗会话。
- 会话真实写入 `PlayerBattleSession` 表，不再由前端自行拼装战斗预览数据。

## POST /api/battle/session/settle

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `result` | `string` | 是 | `victory` 或 `defeat` |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "profile": {
    "account": "PABC1234",
    "currentChapterId": 2,
    "highestUnlockedChapterId": 2
  },
  "progress": {
    "currentChapterId": 2,
    "highestUnlockedChapterId": 2
  },
  "session": {
    "status": "settled",
    "result": "victory"
  }
}
```

### 说明

- `victory` 会按 README 当前章节流转规则，把下一章解锁并切为当前章节。
- `defeat` 只会结算当前会话，不推进章节解锁。
