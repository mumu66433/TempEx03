# 战斗接口

本文档覆盖当前 V0 原型验证阶段可直接驱动前端文字战斗页的后端接口。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/battle/session` | 读取当前账号的战斗会话 |
| `POST` | `/api/battle/session/start` | 基于当前章节创建或重置战斗会话 |
| `POST` | `/api/battle/session/simulate` | 由服务端按 Mission 波次真实模拟整场战斗，返回文字日志、胜负和摘要 |
| `POST` | `/api/battle/session/settle` | 结算当前战斗会话，并将服务端计算出的结果回流到玩家章节进度 |

## 通用说明

- 当前战斗为“整章连战”模型：`start` 只创建会话，`simulate` 会按当前章节所有 Mission 波次连续计算。
- 胜负、回合日志、收益摘要均由服务端生成，前端不再自己推演或拼装结果。
- `settle` 会再次按同样规则用服务端重算一遍结果后再写入进度，因此即使前端漏调 `simulate`，也不会由前端决定胜负。
- 当前实现暂不新增数据库字段，模拟结果会暂存在现有 `PlayerBattleSession.result` 中，直到 `settle` 把会话改为 `settled`。

## POST /api/battle/session/start

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `chapterId` | `number` | 否 | 目标章节；为空时使用玩家当前章节 |

### 行为说明

- 会根据当前章节的首个 `Mission` 初始化 `PlayerBattleSession`。
- 会重置旧的模拟结果：`status = active`、`result = null`、`settledAt = null`。
- 当前接口只负责“进入战斗”，不返回完整战斗日志。

### 成功响应

```json
{
  "ok": true,
  "profile": {
    "account": "PABC1234",
    "currentChapterId": 1,
    "highestUnlockedChapterId": 1
  },
  "session": {
    "chapterId": 1,
    "status": "active",
    "result": null,
    "statusText": "战斗会话进行中",
    "display": {
      "title": "银杏村 战斗准备",
      "subtitle": "第1层 · 第1波 · 普通波"
    },
    "actions": {
      "canStart": false,
      "canSimulate": true,
      "canSettle": true,
      "canRetry": true
    }
  }
}
```

## POST /api/battle/session/simulate

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |

### 服务端模拟规则

- 英雄快照来源于默认角色配置 `Role`，并结合玩家当前最高解锁章节的推荐生命 / 攻击做战力推导。
- 敌人波次来源于当前章节全部 `Mission` 行，按层数和波次顺序连续模拟。
- 每回合先由英雄攻击，再由存活敌人反击。
- 英雄会按攻击速度触发固定节奏的“连击”，按暴击率触发固定节奏的“暴击”。
- 波次之间会有少量生命恢复，用于形成连续战斗节奏。
- 若单波超过上限回合仍未清场，服务端直接判定本场失败，避免前端陷入无限战斗。

## POST /api/battle/session/settle

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |

### 行为说明

- 服务端会再次按当前章节重新模拟一次，确保最终写库结果不依赖前端。
- `victory` 会推进并解锁下一章节；`defeat` 只会结束本次会话，不推进章节。
- 结算后当前会话状态改为 `settled`。

  },

    "result": "victory",
    "currentChapter": {
      "id": 2,

    },

## 联调建议流程

1. 前端进入章节后先调用 `POST /api/battle/session/start`。
2. 点击“开始战斗”后调用 `POST /api/battle/session/simulate`，直接展示 `roundLogs`、`result`、`summaryText`。
3. 玩家确认结果后调用 `POST /api/battle/session/settle`，再刷新首页或章节页进度。
