# 战斗接口

本文档覆盖当前为 V0 前端骨架补齐的最小战斗会话接口。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/battle/session` | 读取当前账号的战斗会话 |
| `POST` | `/api/battle/session/start` | 为当前账号创建或重置最小战斗会话 |
| `POST` | `/api/battle/session/settle` | 结算当前战斗会话，并回写章节进度 |

## GET /api/battle/session

### 请求参数

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | query | 是 | 玩家账号 |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "session": {
    "id": "cm123",
    "chapterId": 1,
    "layerIndex": 1,
    "waveIndex": 1,
    "missionId": 1,
    "waveType": "普通波",
    "enemyId": 1,
    "enemyName": "鸡",
    "enemyCount": 1,
    "enemyLife": 3,
    "enemyAtk": 2,
    "status": "active",
    "result": null,
    "chapter": {
      "id": 1,
      "name": "银杏村",
      "title": "第1章 · 银杏村"
    },
    "statusText": "战斗会话进行中",
    "display": {
      "title": "银杏村 战斗准备",
      "subtitle": "第1层 · 第1波 · 普通波",
      "enemySummary": "鸡 x 1",
      "enemyStats": {
        "life": 3,
        "atk": 2
      }
    },
    "actions": {
      "canStart": false,
      "canSettle": true,
      "canRetry": true
    }
  }

```

### 说明

- 若玩家尚未开始任何战斗，会返回 `"session": null`。
- `statusText`、`display`、`actions` 为后端整理好的战斗页展示字段。

## POST /api/battle/session/start

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `chapterId` | `number` | 否 | 目标章节；为空时使用玩家当前章节 |

### 说明

- 会根据当前章节的首个 `Mission` 配置生成最小战斗会话。
- 会话真实写入 `PlayerBattleSession` 表，不再由前端自行拼装战斗预览数据。
- 返回体中的 `session.display` 和 `session.actions` 可直接驱动战斗页标题、摘要和按钮状态。

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "profile": {
    "account": "PABC1234",
    "currentChapterId": 1,
    "highestUnlockedChapterId": 1
  },
  "session": {
    "chapterId": 1,
    "status": "active",
    "statusText": "战斗会话进行中",
    "display": {
      "title": "银杏村 战斗准备",
      "subtitle": "第1层 · 第1波 · 普通波"
    },
    "actions": {
      "canStart": false,
      "canSettle": true,
      "canRetry": true
    }
  }
}
```

## POST /api/battle/session/settle

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `result` | `string` | 是 | `victory` 或 `defeat` |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "profile": {
    "account": "PABC1234",
    "currentChapterId": 2,
    "highestUnlockedChapterId": 2
  },
  "progress": {
    "currentChapterId": 2,
    "highestUnlockedChapterId": 2
  },
  "settlement": {
    "result": "victory",
    "currentChapter": {
      "id": 2,
      "name": "破庙外道"
    },
    "justUnlockedChapter": {
      "id": 2,
      "name": "破庙外道"
    },
    "nextAction": "返回章节页并刷新到最新解锁进度"
  },
  "session": {
    "status": "settled",
    "result": "victory",
    "statusText": "本局已通关结算",
    "display": {
      "title": "破庙外道 战斗准备",
      "subtitle": "结算结果：victory"
    },
    "actions": {
      "canStart": true,
      "canSettle": false,
      "canRetry": true
    }
  }
}
```

### 说明

- `victory` 会按 README 当前章节流转规则，把下一章解锁并切为当前章节。
- `defeat` 只会结算当前会话，不推进章节解锁。
- 结算后会返回 `settlement` 摘要，方便前端直接展示“解锁了哪一章、下一步该做什么”。
