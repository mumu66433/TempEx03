# 系统与配置接口

本文档覆盖健康检查接口和章节配置读取接口，对应代码位于 `server/src/routes/healthRoutes.js`、`server/src/routes/configRoutes.js` 与相关 service。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/health` | 返回服务健康状态与运行时信息 |
| `GET` | `/api/config/chapter` | 返回章节配置列表 |

## GET /api/health

### 用途

- 用于本地调试、部署验证或简单存活检查。

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "time": "2026-04-25T00:00:00.000Z",
  "uptime": 12.345
}
```

### 说明

- `time` 为服务端当前时间的 ISO 字符串。
- `uptime` 为 Node 进程已运行秒数。

## GET /api/config/chapter

### 用途

- 从内存中的 Excel 配置缓存读取章节数据。
- 为前端章节展示提供最基础的数据源。

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "chapters": [
    {
      "id": 1,
      "chapter": "第1章",
      "name": "银杏村",
      "missionId": 1,
      "missionCount": 5,
      "guessHeroLife": 100,
      "guessHeroAtk": 10,
      "normalWaveRule": "2,3,4",
      "bossWaveCount": 1,
      "waveCycle": "2 3 4",
      "totalWaveEstimate": 9,
      "skillPoolRange": [
        "1",
        "8"
      ],
      "enemyTheme": [
        "村口",
        "野怪"
      ],
      "waveTemplate": "前两层普通波 / 第三层精英波 / 第四层 BOSS 波",
      "title": "第1章 · 银杏村",
      "description": "第1章 · 银杏村 / 关卡数 5 / 预计波次 9 / 推荐生命 100 / 推荐攻击 10"
    }
  ],
  "count": 1
}
```

### 失败响应

HTTP 状态码：`500`

```json
{
  "ok": false,
  "error": "章节配置未加载，请检查 server/src/assets/config/C章节配置表.Chapter.xlsx"
}
```

### 说明

- 当前只暴露章节配置聚合结果，未提供原始工作表透传接口。
- 字段值由 `chapterConfigService` 从 `C章节配置表.Chapter.xlsx` 的首个工作表读取并转换。
- `title` 和 `description` 是后端拼好的展示摘要，章节页可直接使用。
- `skillPoolRange`、`enemyTheme` 会统一转成数组，避免前端再做二次拆分。
