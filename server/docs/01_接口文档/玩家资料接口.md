# 玩家资料接口

本文档覆盖当前 V0 范围内与玩家资料、章节进度相关的接口。

## 接口概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/player/home` | 读取游戏入口页/章节页可直接使用的主页聚合数据 |
| `GET` | `/api/player/profile` | 按账号读取玩家基础资料与章节进度 |
| `GET` | `/api/player/chapters` | 按账号读取带解锁状态的章节列表 |
| `GET` | `/api/player/skills` | 按账号读取功法列表只读数据 |
| `PATCH` | `/api/player/current-chapter` | 更新当前选中的章节 |

## GET /api/player/home

### 请求参数

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | query | 是 | 玩家账号 |

### 返回内容

- `profile`：玩家基础资料与章节进度
- `role`：当前默认角色配置
- `currentChapter`：当前章节摘要
- `chapterOverview`：章节总览统计
- `skillSummary`：功法数量统计
- `battleSession`：当前战斗会话预览
- `display`：给游戏入口页 / 章节页直接使用的展示摘要

### 成功响应

HTTP 状态码：`200`

    "name": "银杏村",
    "title": "第1章 · 银杏村",
    "description": "第1章 · 银杏村 / 关卡数 5 / 预计波次 9 / 推荐生命 100 / 推荐攻击 10"
  },
  "chapterOverview": {
    "currentChapterId": 1,
    "highestUnlockedChapterId": 1,
    "totalChapters": 3,
    "currentChapterTitle": "第1章 · 银杏村"
  },

### 说明

- `chapterOverview.currentChapterTitle` 和 `display` 内字段为后端拼好的展示摘要。
- 玩家还未开始战斗时，`battleSession` 会返回 `null`。

## GET /api/player/skills

### 请求参数

| 字段 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | query | 是 | 玩家账号 |

### 成功响应

HTTP 状态码：`200`

```json
{
  "ok": true,
  "player": {
    "account": "PABC1234",
    "currentChapterId": 1,
    "highestUnlockedChapterId": 1
  },
  "summary": {
    "total": 24,
    "unlocked": 9,
    "owned": 0,
    "upgradeable": 0
  },
  "filters": {
    "grades": ["N", "R", "SR"],
    "states": ["upgrade", "locked"]
  },
  "skills": [
    {
      "id": 1,
      "name": "轻身术",
      "grade": "N",
      "sectType": "通用",
      "moldType": "轻功",
      "unlockChapter": 1,
      "unlocked": true,
      "owned": false,
      "metaText": "N / 通用 / 轻功",
      "statusText": "章节已开放 · 尚未拥有",
      "powerRateText": ["10%", "20%", "50%"],
      "upgradeCondText": ["10", "20"],
      "level": 0,
      "stars": 0
    }
  ]
}
```

### 说明

- 功法静态信息来自 `Skill` 配置表。
- 玩家动态功法数据来自 `PlayerSkillProgress` 表；当前若玩家还未获得功法，则列表会以只读配置态返回。
- `metaText`、`statusText`、`powerRateText`、`upgradeCondText` 这类字段用于前端直接渲染，不必在页面里重复拼接文案。

## PATCH /api/player/current-chapter

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `account` | `string` | 是 | 玩家账号 |
| `chapterId` | `number` | 是 | 目标章节 ID |

### 成功响应

HTTP 状态码：`200`

返回结构与 `GET /api/player/profile` 一致。

### 错误语义

- `账号不能为空`
- `账号不存在`
- `chapterId 必须是正整数`
- `章节不存在`
- `目标章节尚未解锁`
