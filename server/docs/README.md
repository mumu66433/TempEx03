# server/docs

`server/docs` 是程序制作代理维护的后端文档根目录，用于沉淀接口说明、实现约束和配置表规则。

## 目录规划

```txt
server/docs/
  README.md
  01_接口文档/
    README.md
    账号接口.md
    系统与配置接口.md
  02_实现规范/
    README.md
    服务端目录结构.md
    配置加载规范.md
    数据与持久化规范.md
  03_配置表说明/
    README.md
    当前配置表清单.md
```

## 使用约定

- `01_接口文档` 只记录已经存在或明确排期中的后端接口，不在这里写纯前端约定。
- `02_实现规范` 记录服务端实现方式、目录职责、开发约束和后续扩展边界。
- `03_配置表说明` 记录 `server/src/assets/config` 下 Excel 配置表的用途、命名、字段组织和接入约定。
- 新增后端能力时，优先补齐对应分类文档，再推进代码实现，避免实现和约定脱节。
- 文档命名优先保持稳定；新增专题时，优先在对应目录内追加专题文档，不随意改动一级目录名称。

## 当前后端基线

- 服务框架：`Node.js + Express`
- 数据访问：`Prisma + SQLite`
- 配置来源：`server/src/assets/config` 下的 Excel 工作簿
- 当前接口范围：健康检查、账号注册、账号登录、章节配置读取
- 当前启动入口：`server/src/index.js`
