## 命名观察

- 当前工作簿名普遍采用“中文名 + 英文主题后缀”的形式，例如 `C章节配置表.Chapter`。
- 文件名前缀存在按业务类型或模块字母分类的趋势，例如 `C`、`E`、`M`、`R`、`S`。
- 后续新增配置表时，建议继续保持相同命名风格，避免同类资源命名不一致。

## 接入建议

- 服务端正式消费某张配置表前，应先确认字段名、字段类型和首个工作表结构符合 `excelLoader` 约定。
- 如果一张配置表会被多个 service 共同使用，建议新增专门的配置访问 service，避免在业务代码中重复做字段转换。
- 当配置表字段发生增删改时，应同步更新对应接口文档或实现规范中的说明。
