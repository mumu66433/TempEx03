## 5. 功法页是否与 V0 UI 设计语言一致

明确结论：**不一致**。

原因：

- `SkillScene` 当前从 `client/src/utils/ui.js` 引入 `createAvatar`、`createBottomTabs`、`createButton`、`drawRoundedPanel`、`makeLabel`、`makeThemeText`，没有使用 `client/src/utils/v0ui.js`。
- 当前功法页是深色左右分栏布局，视觉更接近旧 UI；V0 设计要求为轻武侠、浅色卡片、金色主按钮、统一底部页签和功法卡层级。
- 当前功法详情是右侧内嵌详情，不是 V0 设计稿中的详情浮层。
- 当前筛选项包含 `可升级`、`未解锁`，功能覆盖有价值，但视觉状态未按 V0 的品质色、卡片置灰、满星角标、可升级角标统一呈现。

建议：功法页应优先整体迁移到 `v0ui.js`，并补充 `createV0SkillCard`、`createV0SkillFilterTabs`、`createV0SkillDetailOverlay` 等组件，避免继续扩大旧 `ui.js` 的使用范围。

## 6. 三选一当前实现状态与缺口

明确结论：**三选一当前未实现**。

当前缺少内容：

| 类型 | 缺口 |
| --- | --- |
| 后端 / API | 缺少“生成三选一候选功法”“刷新一次候选”“确认选择功法”“返回下一层 / 下一波战斗会话”的明确接口与字段契约。当前 `api.js` 未发现三选一相关请求函数。 |
| 前端数据状态 | 缺少三选一候选列表、当前选中项、刷新次数、满星不可选、提交中、提交失败重试、本局功法池收束提示等状态管理。 |
| 前端 UI | 缺少 `V0_04_功法三选一弹窗.svg` 对应的弹窗、3 张卡同屏、选中高亮、确认按钮禁用 / loading、刷新按钮置灰、异常卡置灰。 |
| 战斗流程接入 | `BattleScene` 当前战报播放完成后直接提交结算并进入 `ResultScene`，没有普通层胜利后停顿并打开三选一、确认后继续下一层的分支。 |
| 设计 / 字段对接 | 正式 UI 对接单已明确三选一必做状态、文案边界和验收口径，但仍缺少后端字段映射；候选功法字段、星级 / 等级 / 门派 / 流派 / 效果摘要的数据来源需要继续补齐。 |

建议修复顺序：

1. PM / 后端 / UI 先确认三选一接口契约和正式字段映射。
2. 前端新增 V0 三选一弹窗组件与本地状态。
3. `BattleScene` 在普通层胜利后接入弹窗，BOSS 胜利仍直接结算。
4. 补齐刷新、确认选择、提交失败、满星不可选和 20 本收束提示。
