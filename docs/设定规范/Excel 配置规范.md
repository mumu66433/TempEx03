# Excel 配置规范

## 存放位置

- 将所有游戏数据配置 Excel 文件放在 `server/src/assets/config/` 目录。

## 文件结构

每个配置表使用一个独立的 `.xlsx` 文件，文件名应表达配置类型，
命名规则：
- 首字符：英文配置名首字母大写
- 中文备注：紧接首字符之后
- 分隔符：`.`
- 英文配置名
- 文件后缀

例如：
- `C角色信息表.Characters.xlsx`
- `W武器配置表.Weapons.xlsx`
- `L关卡配置表.Levels.xlsx`

每个文件只需要一个工作表，默认读取第一个工作表内容。

## 数据存储规范

- 前端仅保留账号相关数据
- 所有和游戏相关的数据都存储在服务端
- 前端不保留游戏数据
- 前端需要展示页面时，先向后端请求数据，再组装页面后显示

## 表头规则

表格前 3 行必须固定格式：
1. 第1行：字段名行,单元格格式为 背景色为"#84b6e8"，黑色字，文字上下左右居中
   - 第一个字段通常为 `Id` 或 `intId`，代表主键
   - 其它字段名建议使用驼峰或标准英文单词
2. 第2行：字段备注行,单元格格式为 背景色为"#84b6e8"，黑色字，文字上下左右居中
   - 对应字段的含义说明
3. 第3行：字段类型行,单元格格式为 背景色为"#84b6e8"，黑色字，文字上下左右居中
   - 支持类型：`int`、`number`、`string`、`json`、`array`、`any`、`intId`
   - `intId` 用于主键字段，且该列值必须唯一

## 表格主体内容
1. 如果是首次创建，则需要每个单元格都上下左右居中
2. 非首次创建，即修改表格，则默认当前表格格式

## 数据类型说明

- `int`
  - 必须是整数
- `number`
  - 支持小数
- `string`
  - 按字符串处理
- `json`
  - 如果单元格内容是 JSON 字符串，会解析为对象或数组
- `array`
  - 可以是 JSON 数组，也可以是逗号分隔列表
- `any`
  - 无类型限制，直接保留原始值
- `intId`
  - 与 `int` 语义类似，但会额外检查唯一性

## 外键引用规则

- 任何以 `Id` 结尾的字段都会尝试作为外键解析
- 例如：`defaultWeaponId` 会自动关联到 `W武器配置表.Weapons.xlsx` 的主键行
- 如果字段值为逗号分隔字符串，则会解析成多个引用对象数组

## 校验规则

加载时会进行以下检查：

- 字段类型行必须存在
- `int` / `intId` 字段必须是整数
- `number` 字段会按数字解析，支持整数和小数
- `json` 字段必须是合法 JSON
- `array` 字段必须能解析成数组
- `intId` 字段值不能重复

## 百分比字段约定

- 需要表达百分比时，建议直接按小数存储
- 例如 `10%` 建议在 Excel 中填写为 `0.1`
- 当前解析器不会自动把带 `%` 的文本转换成小数

## 版本与兼容

- 该规范适用于当前 `server/src/config/excelLoader.js` 解析器
- 如果后续需要兼容更多类型，可在此文档中扩展类型定义
