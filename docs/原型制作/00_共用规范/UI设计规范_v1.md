# 剑侠风云 UI 设计规范 v1

本规范用于指导 `剑侠风云` 微信小游戏的 UI 与 UE 设计，后续版本默认在本规范基础上扩展。

## 1. 设计目标

- 竖屏单手操作优先，信息布局必须适合 `750 x 1624` 设计尺寸。
- 风格方向以“轻武侠、清爽、可读”为主，不做过度厚重或玄幻化的界面。
- 首屏要快懂，战斗页要快读，三选一弹窗要快选，章节页要快进。
- 界面在原型阶段先保证层级清晰，再逐步补插画、特效和材质细节。

## 2. 设计尺寸

- 设计基准尺寸：`750 x 1624`
- 安全边距：左右各 `32`，顶部主内容起始建议不低于 `96`，底部主内容结束建议不低于 `128`
- 栅格建议：`12` 列逻辑栅格，常用内容宽度为 `686`
- 主按钮高度：`88`
- 次按钮高度：`72`
- 顶部标题栏高度：`88`
- 底部页签栏高度：`120`
- 常规弹窗宽度：`622`
- 卡片圆角：
  - 页面大卡片 `28`
  - 弹窗与列表卡 `24`
  - 小按钮与状态标签 `18`

## 3. 颜色规范

### 3.1 主色

- 主背景深色：`#0F172A`
- 主背景渐变起点：`#13233C`
- 主背景渐变终点：`#1F4F78`
- 主强调金色：`#F4C56A`
- 辅助青色：`#7FD6C2`

### 3.2 中性色

- 页面底板：`#F5F0E6`
- 卡片底色：`#FFF9F0`
- 描边浅色：`#D8CBB3`
- 标题主字：`#1F2937`
- 正文文字：`#475467`
- 弱提示文字：`#98A2B3`

### 3.3 功能色

- 成功/通关：`#5CBF75`
- 危险/失败：`#D96B6B`
- 警告/提示：`#F2A94A`
- 选中高亮：`#FFE7A8`

### 3.4 品质色

- N：`#9AA4B2`
- R：`#4C89D9`
- SR：`#8A63D2`
- SSR：`#D99A3D`
- UR：`#D94F5C`

## 4. 字体与层级

- 默认字体：优先系统无衬线，标题可在正式视觉阶段替换为书法感标题字。
- 页面标题：`44 / semibold`
- 模块标题：`32 / semibold`
- 关键数值：`36 / bold`
- 正文：`26 / regular`
- 辅助说明：`22 / regular`
- 标签与按钮文案：`24 / semibold`

## 5. 组件规范

### 5.1 按钮

- 主按钮：
  - 深蓝底 + 金色描边
  - 用于“开始战斗”“确认选择”“继续”
- 次按钮：
  - 浅底 + 深色字
  - 用于“返回”“关闭”“切换账号”
- 危险按钮：
  - 红色系底
  - 只用于放弃、退出等风险操作

### 5.2 卡片

- 技能卡、章节卡、结算卡统一采用浅色底板 + 明显描边
- 卡片必须保留标题区、核心信息区、状态区三个层级
- 技能卡优先展示：
  - 功法名
  - 品质
  - 门派 / 流派
  - 当前等级或星级
  - 核心收益一句话

### 5.3 弹窗

- 弹窗必须包含：
  - 标题
  - 主体内容
  - 主操作按钮
  - 关闭或取消入口
- 三选一弹窗要支持横向 3 卡平铺，默认单屏可完整看完

### 5.4 底部导航

- V0 阶段保留 `章节 / 功法 / 角色 / 功法商城` 4 个页签位
- 实际开放只做 `章节` 与 `功法`
- 未开放页签保留骨架，不做可点击高亮态

## 6. 交互逻辑规范

### 6.1 页面跳转

- 所有主流程跳转优先使用“当前页推进”，减少层级过深
- V0 核心路径：
  - 登录页 -> 章节页 -> 战斗页 -> 三选一 -> 战斗页 -> 结算页 -> 章节页

### 6.2 动效建议

- 页面切换：`200ms - 280ms` 淡入/位移
- 弹窗出现：`180ms` 缩放 + 淡入
- 卡片选中：描边高亮 + 轻微上浮
- 按钮点击：轻缩放反馈

### 6.3 交互优先级

- 第一优先：可读性
- 第二优先：点击反馈
- 第三优先：武侠氛围
- 原型阶段不追求复杂转场和重粒子效果

## 7. V0 页面范围

V0 原型验证必须覆盖以下页面：

- 登录/游戏入口页
- 章节选择页
- 战斗页 HUD
- 功法三选一弹窗
- 功法列表页
- 功法详情浮层
- 通关结算页
- 失败结算页

## 8. 研发对接原则

- 先按 `750 x 1624` 进行 UI 切图和布局开发，客户端再做等比缩放适配。
- 设计图中的示例文案、示例头像、示例图标，原型阶段允许使用占位资源。
- 状态稿优先于单一静态稿，至少要区分：
  - 默认态
  - 选中态
  - 禁用态
  - 通关 / 失败态

## 9. 后续维护

- 新版本如果沿用现有设计语言，只在对应版本目录增加页面与状态稿即可。
- 如果修改了颜色、尺寸、导航结构、按钮标准，必须先更新本规范，再更新版本原型。
