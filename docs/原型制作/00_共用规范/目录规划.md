# 原型制作目录规划

`docs/原型制作` 是设计侧的根目录，用于沉淀 UI 规范、组件资产、版本原型和交互说明。

## 目录结构

```txt
docs/原型制作/
  00_共用规范/                  设计规范、命名规范、尺寸规则、颜色与交互标准
  01_组件示例/                  共用组件、样式板、通用 UI 示例
  版本原型/
    V0_原型验证/
      00_说明/                  版本目标、适用范围、设计说明
      01_流程与结构/            页面流转、信息架构、低保真交互说明
      02_界面设计/              该版本的页面设计稿、界面示例、状态稿
      03_切图与标注/            尺寸标注、切图说明、研发对接备注
    V1_首发版本/                后续版本沿用同样结构
    V1.1_内容版本/
    V1.2_社交版本/
```

## 使用规则

- `00_共用规范` 只维护跨版本共用的设计规则，不写版本专属业务逻辑。
- `01_组件示例` 只放通用组件和视觉样板，例如按钮、卡片、弹窗、底部导航。
- `版本原型` 下每个版本必须单独建目录，禁止把不同版本的页面混放。
- 版本目录内的页面设计、交互说明、研发标注都必须跟着该版本走，后续调整也优先在对应版本目录更新。
- 如果共用规范调整影响多个版本，需要先更新 `00_共用规范`，再决定是否同步修正历史版本稿。
- 文件命名建议按“版本_模块_页面_状态”命名，例如 `V0_章节界面_默认态.svg`。
