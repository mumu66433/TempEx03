# V0 前端 UI 对接单

本文档为 V0 原型验证阶段的正式前端 UI 开发依据，用于对齐 `docs/原型制作/版本原型/V0_原型验证` 下的界面设计稿、状态稿、标注说明与前端实现。

## 1. 使用范围

- 设计基准：`750 x 1624` 竖屏画布，前端按等比缩放适配。
- 主要参考：
  - `02_界面设计/V0_01_登录界面.svg`
  - `02_界面设计/V0_02_章节界面.svg`
  - `02_界面设计/V0_03_战斗界面.svg`
  - `02_界面设计/V0_04_功法三选一弹窗.svg`
  - `02_界面设计/V0_05_功法列表界面.svg`
  - `02_界面设计/V0_06_功法详情浮层.svg`
  - `02_界面设计/V0_07_通关结算界面.svg`
  - `02_界面设计/V0_08_失败结算界面.svg`
  - `02_界面设计/V0_09_战斗状态稿.svg`
  - `02_界面设计/V0_10_功法状态稿.svg`
  - `03_切图与标注/V0_界面标注说明.md`
  - `03_切图与标注/V0_章节页状态补充说明.md`
- 账号登录 / 注册表单沿用当前项目现有弹层，不在本对接单内重做输入表单 UI。
- 游戏主界面可由 Phaser 3 场景、DOM 覆盖层或两者组合实现；实现方式可调整，但最终视觉结构、层级、状态与交互必须符合本文档。

## 3. 全局状态规则

| 状态 | 全局表现规则 | 禁止行为 |
| --- | --- | --- |
| 默认 | 展示真实接口数据或明确的设计示例结构；主按钮可点击 | 不使用无来源的假数据冒充真实数据 |
| 加载中 | 页面主内容区显示骨架、加载文案或轻量遮罩；相关主操作禁用 | 不允许在加载中重复发起相同主请求 |
| 请求失败 | 显示失败标题、原因文案和重试按钮；保留可返回路径 | 不允许静默失败或停留空白页 |
| 空数据 | 显示明确空态标题、说明和可恢复动作 | 不渲染占位假卡片误导用户 |
| 禁用 | 按钮置灰、卡片降透明度，并给出原因提示 | 禁用控件不得触发主流程跳转 |
| 选中 | 卡片描边高亮、轻微上浮或底色变化；同组单选只能有一个选中项 | 不允许仅靠颜色区分且无结构反馈 |
| 提交中 / 处理中 | 主按钮进入 loading 或禁用，页面保留当前选择和结果 | 不允许重复提交、重复领取、重复开始 |
| 未开放提示 | 统一轻提示文案：`该功能 V0 暂未开放` 或更具体的未开放原因 | 不进入空白页面，不跳到未实现路由 |

## 4. 逐页状态要求

### 4.1 游戏入口页

| 状态 | UI 要求 | 交互要求 |
| --- | --- | --- |
| 默认 | 展示游戏标题、版本提示、主按钮 `进入游戏`、次按钮 `切换账号` | 点击进入章节页；切换账号回到现有账号弹层 |
| 加载中 | 登录态校验或账号信息读取期间显示 `登录状态确认中` | 禁用进入游戏 |
| 请求失败 | 显示 `登录状态异常`、说明 `请重新登录后进入游戏` | 提供 `重新登录` |
| 空数据 | 若账号缺少昵称或角色信息，显示 `少侠`、`--` 等空值文案 | 仍允许进入章节页，数据以后端返回为准 |
| 禁用 | 未完成登录态确认时主按钮禁用 | 点击禁用按钮给轻提示 |
| 选中 | 无复杂选中态 | 不需要额外状态稿 |
| 提交中 / 处理中 | 点击进入游戏后按钮显示 `进入中` | 防止重复点击 |
| 未开放提示 | 入口页不承载未开放模块 | 不显示多余未开放入口 |

### 4.2 章节页

| 状态 | UI 要求 | 交互要求 |
| --- | --- | --- |
| 默认 | 顶部角色信息、中部章节卡、底部 `章节 / 功法 / 角色 / 商城` 页签 | 当前可挑战章节主按钮为 `开始战斗` |
| 加载中 | 章节卡区域显示 2-3 张骨架卡或 `章节加载中` | `开始战斗` 禁用；已开放页签可切换 |
| 请求失败 | 显示 `章节加载失败`、`请检查网络或稍后重试`、`重试` | 重试重新拉取章节数据 |
| 空数据 | 显示 `暂无章节配置`、`请检查服务端章节配置后重试` | 不进入战斗页 |
| 禁用 | 未解锁章节卡置灰，按钮置灰 | 点击提示 `通关上一章后解锁` |
| 选中 | 已解锁章节可切换预览；当前章节显示 `当前章节` 标签 | 选中后主按钮指向选中章节 |
| 提交中 / 处理中 | 点击开始战斗后主按钮显示 `进入中` | 防重复创建战斗 |
| 未开放提示 | `角色`、`商城` 页签保留骨架位 | 点击提示 `该功能 V0 暂未开放` |

### 4.3 战斗页

| 状态 | UI 要求 | 交互要求 |
| --- | --- | --- |
| 默认 | 顶部 HUD、中心文字战报、底部构筑栏三段式结构 | 自动推进普通层、精英层、Boss 层 |
| 加载中 | 战斗初始化显示 `战斗准备中` 或中心区 loading | 暂停、返回等次操作可按实现成本保留 |
| 请求失败 | 显示 `战斗加载失败` 和 `返回章节` / `重试` | 重试重新请求战斗模拟或恢复流程 |
| 空数据 | 若无战报，中心区显示 `暂无战报` | 不渲染空白日志列表 |
| 禁用 | 自动战斗推进中禁用重复开始；结算处理中禁用返回结算按钮 | 暂停态按弹层规则处理 |
| 选中 | 底部功法卡可显示当前触发或当前主功法高亮 | 不影响自动战斗节奏 |
| 提交中 / 处理中 | 进入结算前显示 `结算中` 遮罩或按钮 loading | 防止重复提交结算 |
| 未开放提示 | 图形化人物演出、复杂粒子特效 V0 不开放 | 不因占位特效影响战报可读性 |

### 4.4 功法三选一弹窗

| 状态 | UI 要求 | 交互要求 |
| --- | --- | --- |
| 默认 | 半透明遮罩 + 弹窗主体 + 3 张功法卡 + `确认选择` | 未选中前确认按钮禁用 |
| 加载中 | 刷新候选时 3 卡进入骨架或禁用态 | 清空当前选中 |
| 请求失败 | 显示 `功法刷新失败` 和 `重试`；保留原候选时需提示数据未更新 | 不关闭弹窗 |
| 空数据 | 显示 `暂无可选功法`，可提供 `继续战斗` 兜底 | 兜底需由前后端联调确认 |
| 禁用 | 满星、无效或异常卡片置灰并标注原因 | 点击只给提示，不进入选中态 |
| 选中 | 选中卡描边高亮，上浮或底色变化 | 只允许单选 |
| 提交中 / 处理中 | 确认后按钮显示 `选择中`，所有卡片禁用 | 成功后回到战斗页下一层 |
| 未开放提示 | 非调试模式不提供跳过；刷新次数用尽显示 `已刷新` | 调试跳过必须标注 `调试跳过` |

## 5. 主流程流转

| 流程 | 触发 | 页面变化 | 失败兜底 |
| --- | --- | --- | --- |
| 登录后入口 | 账号登录 / 自动登录成功 | 进入游戏入口页 | 登录失败沿用现有账号弹层错误提示 |
| 入口到章节 | 点击 `进入游戏` | 进入章节页并加载章节数据 | 显示章节加载失败或登录态异常 |
| 章节到战斗 | 点击可挑战章节 `开始战斗` | 进入战斗页并初始化战斗 | 回到章节页或在战斗页显示重试 |
| 普通层胜利到三选一 | 普通层战斗胜利 | 弹出功法三选一 | 候选异常时显示重试或兜底继续 |
| 三选一回战斗 | 选择功法并确认 | 关闭弹窗，进入下一层 / 下一波战斗 | 保留弹窗并提示选择失败 |
| Boss 胜利到结算 | Boss 层胜利 | 进入胜利结算 | 结算失败保留胜利页并允许重试提交 |
| 失败到结算 | 玩家战败 | 进入失败结算 | 结算失败保留失败页并允许重试提交 |
| 结算回章节 | 点击 `返回章节` | 回到章节页并刷新解锁与进度 | 刷新失败显示章节请求失败状态 |
| 功法列表到详情 | 点击功法卡 | 打开功法详情浮层 | 详情加载失败显示重试和关闭 |

## 6. 文案边界

### 6.1 标题与按钮

| 场景 | 推荐文案 | 边界规则 |
| --- | --- | --- |
| 游戏入口标题 | `剑侠风云` | 标题最多 6 个中文字符，超出需调整字号或分行 |
| 进入游戏按钮 | `进入游戏` | 提交中改为 `进入中` |
| 章节主按钮 | `开始战斗` | 禁用时保留按钮位，文案可改为 `暂未解锁` |
| 三选一确认 | `确认选择` | 未选中时禁用，提交中为 `选择中` |
| 胜利返回 | `返回章节` | 结算中禁用 |
| 失败再试 | `再试一次` | 点击后显示 `进入中` |
| 未开放 | `该功能 V0 暂未开放` | 可按模块改为 `升级功能 V0 暂未开放` |

### 6.2 日志与战报

| 文案类型 | 最大展示 | 换行 / 省略 |
| --- | --- | --- |
| 最新战报标题 | 24 个中文字符 | 最多 2 行，超出省略 |
| 单条回合日志 | 28 个中文字符 | 单行省略；日志列表保留最近 5-7 条 |
| 波次横幅 | 16 个中文字符 | 单行居中，超出缩小字号或省略 |
| 战报摘要 | 60 个中文字符 | 最多 3 行，超出省略并保留关键数值 |

### 6.3 奖励与空值

| 内容 | 空值文案 | 边界规则 |
| --- | --- | --- |
| 章节名 | `未命名章节` | 最多 10 个中文字符，最多 2 行 |
| 推荐战力 | `--` | 数值过长使用万 / 亿缩写 |
| 功法名 | `未知功法` | 最多 8 个中文字符，长名最多 2 行 |
| 奖励名 | `无奖励` | 单项最多 12 个中文字符 |
| 奖励数量 | `x0` | 缺失时不展示数量或显示 `x0`，不可显示 `undefined` |
| 失败原因 | `未能击败本层敌人` | 最多 2 行 |

### 6.4 错误提示

| 章节加载失败 | `章节加载失败` | `请检查网络或稍后重试` |

| 结算失败 | `结算提交失败` | `当前结果已保留，请重试提交` |
| 未开放功能 | `暂未开放` | `该功能 V0 暂未开放` |

## 7. 资源交付与占位规则

### 7.1 必须按设计稿还原

- 750 x 1624 竖屏比例与安全边距。
- 顶部 HUD / 中心主内容 / 底部导航或构筑栏的三段式层级。
- 章节卡、功法卡、弹窗、结算卡的圆角、层级、主次按钮位置。
- 主题色、功能色、品质色的基本语义：
  - 主背景深蓝系。
  - 主强调金色。
  - 成功绿色、失败红色、警告橙色、选中浅金色。
- 三选一弹窗必须同屏完整展示 3 张卡，不做横向滚动。
- 请求失败、空数据、禁用、提交中等状态必须明确可见。

### 7.2 可由代码绘制

## 9. 前端一致性自查报告要求

前端完成 V0 UI 落地后，需要新增一份“当前实现与 UI 设计稿一致性自查报告”，并将本文档作为对照依据之一。报告建议放在前端或项目约定的协作目录，由前端代理按实际实现路径决定；报告必须至少覆盖：

| 检查项 | 要求 |
| --- | --- |
| 对照依据 | 列出本文档、V0 SVG 设计稿、`V0_界面标注说明.md`、`V0_章节页状态补充说明.md` |
| 页面覆盖 | 逐页列出游戏入口页、章节页、战斗页、三选一、功法列表、详情浮层、胜利结算、失败结算 |
| 状态覆盖 | 标记默认、加载中、请求失败、空数据、禁用、选中、提交中 / 处理中、未开放提示是否已实现 |
| 偏差说明 | 对每个未 1:1 还原项说明原因，归类为工程化调整、V0 占位或待确认 |
| 截图 / 录屏 | 至少提供核心流程截图；若无法提交图片，需列出本地验证页面与操作路径 |
| 待确认项 | 汇总需要 PM / UI / 前端确认的差异，不得混在代码说明中 |

一致性自查结论建议使用以下枚举：

- `一致`：实现与本文档及设计稿一致。
- `工程化调整`：视觉或交互略有调整，但不改变结构、层级、状态和主流程。
- `V0 占位`：按本文档允许占位项处理。
- `待确认`：超出本文档规则，需要 PM / UI / 前端重新确认。
- `未完成`：必做项尚未实现。
