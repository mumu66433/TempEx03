# V0 前端 UI 对接单示例

本文档用于给前端一个可直接执行的 V0 UI 对接样例，目的是说明：

- 你需要提供哪些资料
- 每类资料建议怎么写
- 一份可直接参考的示例内容应该长什么样

本示例按当前 V0 原型验证范围整理，重点覆盖章节页、战斗页、三选一弹窗、功法页和结算页。

## 一、前端真正需要你提供的内容

前端严格按 UI 落地时，至少需要以下 6 类资料：

| 资料类型 | 你要提供什么 | 作用 |
| --- | --- | --- |
| 页面范围清单 | 本期到底做哪些页面，哪些不做 | 防止前端误做超范围页面 |
| 页面设计稿与状态稿 | 每页默认态、加载态、空态、失败态、禁用态 | 防止前端自行猜状态 |
| 交互流转说明 | 点哪里跳哪里，什么时候弹窗，什么时候返回 | 防止实现和原型理解不一致 |
| 未决策项结论 | 原型里还没拍板的地方统一定稿 | 防止前端反复返工 |
| 字段映射表 | UI 上每块内容对应哪个接口字段 | 防止前端猜字段和假写文案 |
| 资源交付包 | 背景、按钮、图标、字体说明、命名规则 | 防止只能做占位样式，无法贴近原型 |

## 二、建议你按什么格式提供

### 1. 页面范围清单

建议格式：

| 页面名 | 本期是否必做 | 入口 | 主要出口 | 备注 |
| --- | --- | --- | --- | --- |
| 游戏入口页 | 是 | 登录成功后 | 进入章节页 | 如果最终取消，需要提前拍板 |
| 章节页 | 是 | 游戏入口页 / 登录成功直达 | 战斗页、功法页 | 当前 V0 核心页面 |
| 战斗页 | 是 | 章节页点击开始战斗 | 三选一、结算页、章节页 | 战斗表现采用文字战报 |
| 三选一弹窗 | 是 | 普通层战斗胜利后 | 返回战斗页 | Boss 层不弹 |
| 功法列表页 | 是 | 章节页底部页签 | 功法详情浮层 | |
| 功法详情浮层 | 是 | 功法列表点击卡片 | 返回功法列表页 | 升级功能是否可点需拍板 |
| 胜利结算页 | 是 | Boss 胜利后 | 返回章节页、继续查看功法 | |
| 失败结算页 | 是 | 战斗失败后 | 再试一次、返回章节页 | |
| 账号登录/注册表单 | 否 | 网页初始打开时 | 登录成功后进入游戏 | 当前沿用现有弹层，不重做 UI |

### 2. 页面状态说明

建议格式：

| 页面名 | 必做状态 | 说明 |
| --- | --- | --- |
| 章节页 | 默认态 | 展示当前章节、已解锁章节、未解锁章节 |
| 章节页 | 加载中 | 请求章节配置时显示骨架或加载文案 |
| 章节页 | 请求失败 | 显示失败提示和重试按钮 |
| 章节页 | 空数据 | 显示暂无章节配置提示，不可进入战斗 |
| 战斗页 | 普通波次 | 展示普通敌人回合战报 |
| 战斗页 | 精英波次 | 文案高亮，战报强调精英来袭 |
| 战斗页 | Boss 波次 | 展示 Boss 来袭和结算前总结 |
| 战斗页 | 暂停态 | 弹出暂停面板 |
| 三选一弹窗 | 未选择 | 确认按钮禁用 |
| 三选一弹窗 | 已选择 | 高亮选中卡片，允许确认 |
| 功法列表页 | 已解锁 | 正常显示功法卡 |
| 功法列表页 | 未解锁 | 卡片置灰，显示解锁条件 |
| 功法列表页 | 可升级 | 显示可升级角标 |
| 结算页 | 胜利结算 | 显示奖励和本局战报摘要 |
| 结算页 | 失败结算 | 显示失败原因和建议按钮 |

### 3. 交互流转说明

建议格式：

| 触发动作 | 页面变化 | 接口请求 | 失败兜底 |
| --- | --- | --- | --- |
| 登录成功 | 进入游戏入口页或章节页 | `POST /api/auth/login` | Toast 提示登录失败 |
| 点击章节页“开始战斗” | 进入战斗页 | `POST /api/battle/session/simulate` | 回到章节页并提示稍后重试 |
| 普通层胜利 | 弹出三选一弹窗 | 无额外请求或使用当前战斗结果 | 若数据异常则直接进入下一层并给提示 |
| Boss 层胜利 | 进入胜利结算页 | 使用战斗结算数据 | 提示结算异常并允许重试 |
| 战斗失败 | 进入失败结算页 | 使用战斗结算数据 | 提示结算异常并允许重试 |
| 点击“返回章节” | 弹出二次确认 | 无 | 取消则留在当前页 |
| 点击“再试一次” | 重新开始当前章节战斗 | `POST /api/battle/session/simulate` | 保留失败页并提示重试失败 |

### 4. 字段映射表

建议格式：

| UI 位置 | 字段名 | 来源接口 | 空值显示 | 最大长度 |
| --- | --- | --- | --- | --- |
| 章节卡标题 | `chapterName` | `GET /api/chapters` | `未命名章节` | 10 字 |
| 章节卡推荐战力 | `recommendedPower` | `GET /api/chapters` | `--` | 8 字 |
| 战斗页当前波次 | `currentWave` / `totalWave` | `POST /api/battle/session/simulate` | `--/--` | 固定格式 |
| 战斗主战报 | `summaryText` | `POST /api/battle/session/simulate` | `暂无战报` | 24 字 |
| 战斗回合日志 | `roundLogs[]` | `POST /api/battle/session/simulate` | 不显示列表 | 单条 28 字 |
| 结算奖励文案 | `rewards[].name` | `POST /api/battle/session/simulate` | `无奖励` | 12 字 |
| 功法卡名称 | `skillName` | 功法列表接口 | `未知功法` | 8 字 |

### 5. 资源交付包

建议格式：

```txt
/ui-export/v0/
  chapter/
    bg_chapter.png
    card_chapter_locked.png
    card_chapter_active.png
  battle/
    bg_battle.png
    panel_battle_log.png
  common/
    btn_primary.png
    btn_primary_disabled.png
    btn_secondary.png
    icon_lock.png
    icon_pause.png
    tab_chapter.png
    tab_skill.png
  font/
    字体说明.md
```

建议说明清楚：

- 是否允许前端用代码绘制替代
- 哪些资源必须 1:1 贴图
- 哪些资源可先占位
- 字体是否有授权和替代方案

### 6. 验收口径

建议格式：

| 页面名 | 必须一致项 | 可容差项 | 允许占位项 |
| --- | --- | --- | --- |
| 章节页 | 顶部信息区、章节卡层级、底部页签位置 | 文案换行可随真实数据微调 | 头像可先用占位图 |
| 战斗页 | 上 HUD、中战报区、下构筑条三段式结构 | 日志滚动速度可工程化调整 | 特效先不做 |
| 三选一弹窗 | 三卡同屏展示、按钮位置和层级 | 文案长度超出时可省略 | 卡面插画可先占位 |
| 结算页 | 奖励区、摘要区、主按钮区布局 | 文案间距可按真实数据微调 | 奖励图标可先占位 |

## 三、完整示例内容

下面是一份可以直接发给前端的简化示例。

### V0 页面范围

`V0 需要做：游戏入口页、章节页、战斗页、三选一弹窗、功法列表、功法详情、胜利结算、失败结算；账号登录/注册表单沿用现有弹层，不重做。`

### V0 页面状态

`章节页需要：默认态、已解锁、未解锁、加载中、请求失败、空数据。`

`战斗页需要：普通波、精英波、Boss 波、暂停态、返回章节二次确认。`

`三选一弹窗需要：默认态、选中态、刷新后 loading 态。`

`功法页需要：已解锁、未解锁、可升级、满星、筛选态。`

`结算页需要：胜利结算、失败结算、结算提交中、结算失败重试。`

### V0 交互流转

`登录成功 -> 进入游戏入口页。`

`点击“开始战斗” -> 进入战斗页。`

`普通层胜利 -> 弹出三选一。`

`确认选择功法 -> 返回战斗页继续下一层。`

`Boss 胜利 -> 直接进入胜利结算。`

`战斗失败 -> 进入失败结算。`

`点击“返回章节” -> 先弹二次确认。`

### V0 未决策项结论示例

请至少把下面几项明确写给前端：

| 待确认项 | 建议写法示例 |
| --- | --- |
| 是否保留游戏入口页 | `V0 保留游戏入口页，登录成功后先进入游戏入口页，再从入口页进入章节页。` |
| 功法详情页升级按钮是否可点 | `V0 功法详情页保留升级按钮样式，但先不接真实升级逻辑，点击后提示“V0 暂未开放”。` |
| 调试入口放在哪里 | `调试入口仅在开发环境显示，放在章节页右上角；正式演示环境隐藏。` |

### V0 字段映射示例

| UI 位置 | 字段名 | 来源 | 说明 |
| --- | --- | --- | --- |
| 章节卡标题 | `name` | 章节接口 | 显示章节名称 |
| 章节卡层数 | `floorTotal` | 章节接口 | 显示总层数 |
| 推荐战力 | `recommendedPower` | 章节接口 | 若无则显示 `--` |
| 战斗主战报 | `summaryText` | 战斗模拟接口 | 用作主标题战报 |
| 战斗日志 | `roundLogs` | 战斗模拟接口 | 渲染最近几条日志 |
| 结算奖励 | `rewards` | 战斗模拟接口 | 渲染奖励列表 |

### V0 文案边界示例

| 项目 | 规则 |
| --- | --- |
| 章节名 | 最多 10 字，超出两行省略 |
| 战报主标题 | 最多 24 字，超出截断 |
| 回合日志 | 单条最多 28 字，超出省略 |
| 错误提示 | 统一使用“请检查网络后重试” |
| 未开放提示 | 统一使用“该功能 V0 暂未开放” |

## 四、如果你现在只想先补最小一套

建议优先给前端这 4 份内容：

1. `页面范围清单`
2. `每页必做状态`
3. `主流程拍板结论`
4. `最小资源包`

最小资源包建议至少包含：

- 章节页背景
- 主按钮默认态和禁用态
- 锁定图标
- 底部页签图标
- 字体使用说明

## 五、你可以直接照抄的提交给前端的话术

你可以直接把下面这段发给前端：

```txt
本期 V0 必做页面为：游戏入口页、章节页、战斗页、三选一弹窗、功法列表、功法详情、胜利结算、失败结算。账号登录/注册继续沿用现有弹层，不重做新 UI。

章节页必须实现默认态、已解锁、未解锁、加载中、失败、空数据；战斗页采用文字战报方案，保留上 HUD、中战报区、下构筑条三段式结构；普通层胜利后弹三选一，Boss 胜利后直接结算；返回章节必须二次确认。

调试入口仅开发环境显示，放章节页右上角。功法详情页升级按钮 V0 先只展示样式，不接真实升级逻辑。资源包后续补齐，当前先按已给 SVG 与说明文档实现，必须保证整体结构和层级贴近原型。
```

## 六、补充说明

- 如果后续新增页面或改主流程，必须先更新本对接单，再让前端改实现。
- 如果只给静态图、不补状态和交互，前端只能做“看起来像”，很难做到“按 UI 准确实现”。
- 如果资源包暂时不齐，建议在文档里明确哪些地方允许前端先占位，避免验收阶段反复重做。
