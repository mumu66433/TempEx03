# V0 界面标注说明

## 标注原则

- 所有页面以 `750 x 1624` 为标准尺寸输出。
- 标注优先写结构尺寸和相对关系，不在原型阶段过度细化像素微调。
- 交互说明要写“点击后去哪里”，而不只写“这是个按钮”。

## 研发优先关注

- 顶部安全区与底部页签区必须预留固定高度。
- 章节页、功法页都采用“上信息 + 中内容 + 下导航”的统一结构。
- 战斗页信息要尽量压缩到上方和下方，中心区域留给战斗表现。
- 三选一弹窗必须一次完整展示 3 张卡，避免横向滚动。
- 游戏主界面由 `Phaser 3` 场景承载，账号登录 / 注册仍沿用现有登录层；游戏内弹窗、HUD 和结算可以按实现成本拆为 Phaser 内绘或 DOM 覆盖层，但视觉层级必须保持一致。
- `V0_01_登录界面.svg` 为登录成功后的游戏入口页，不等于账号表单页；账号输入、注册、自动登录、切换账号逻辑仍走现有登录层。

## V0 推荐输出物

- 页面设计稿：默认态
- 页面设计稿：关键交互态
- 组件示例：按钮 / 卡片 / 弹窗 / 标签
- 标注文档：尺寸、层级、交互路径

## 章节页状态与交互

- 当前可挑战章节：显示“当前章节”标签、章节名、层数 / 波数、推荐战力和主按钮“开始战斗”；点击后进入战斗页。
- 已解锁章节：显示“已解锁”标签，可点击切换为当前预览章节；如直接点击章节卡底部按钮，则进入该章节战斗。
- 未解锁章节：卡片降透明度，按钮置灰，展示“通关上一章后解锁”；点击只给轻提示，不进入战斗。
- 战力不足提示：V0 不强拦截，保留黄色提示“推荐战力不足，仍可挑战”，按钮仍可点击。
- 章节通关回流：通关结算点击“返回章节”后，章节页刷新解锁状态，并把下一章标为当前可挑战。
- 调试入口：仅原型 / 开发环境展示在章节页右上角小按钮“调试”，线上隐藏。
- 未开放页签：`角色`、`商城` 仅保留骨架位，不高亮、不跳转；点击时给轻提示“暂未开放”。
- 接口态：
  - 加载中：章节卡区域显示骨架或“章节加载中”提示，主按钮禁用。
  - 请求失败：章节卡区域显示失败提示和“重试”按钮，底部页签仍可正常切换到已开放页。
  - 空数据 / 无有效配置：显示“暂无章节配置，请检查服务端配置”提示，不进入战斗页。

## 战斗页状态与边界

### Phaser 场景负责

- 战斗背景、我方角色、敌人单位、弹道 / 近战动作、受击飘字、死亡淡出、波次入场、BOSS 入场压迫表现。
- 自动战斗循环中的站位、攻击节奏、受击反馈、波次切换和死亡判定。

### DOM / 覆盖层 UI 负责

- 顶部章节 / 层 / 波信息、我方与敌方血条、当前构筑条、暂停按钮、调试浮层。
- 普通层通关后的功法三选一弹窗、暂停浮层、通关 / 失败结算页。
- 如果程序选择全部用 Phaser 绘制，也需按同一层级保留“顶部状态、中心战斗、底部构筑”的信息分区。

### 必备状态

- 普通波：显示普通波标签、当前波 / 总波、普通怪血条和自动战斗状态。
- 精英波：显示精英波标签，敌方卡片使用更高描边或红色角标，提示压力提升。
- BOSS 层：顶部显示“第5层 BOSS”，中心敌人更大，BOSS 血条横向拉长；BOSS 层胜利后直接进入结算，不再弹三选一。
- 受击：角色或敌人短暂闪白 / 红，显示伤害飘字；血条变化需有 120ms-180ms 过渡。
- 死亡：死亡方降透明度并停止攻击，若我方死亡进入失败结算，若敌方全灭则进入下一波或层结算。
- 波次切换：中心显示“第 N 波完成 / 下一波来袭”，短暂停留 600ms-900ms 后继续自动战斗。
- 暂停：点击“暂停”后显示半透明遮罩和暂停面板，提供“继续战斗”“返回章节”。
- 返回章节确认：点击“返回章节”后弹出二次确认“返回章节将结束本局挑战，是否继续”；确认后退出本局回到章节页，取消则回到暂停面板。
- 底部构筑条：单屏优先展示当前排序靠前的功法卡；当数量超出单屏宽度时，使用横向滚动查看，不额外新开列表页。
- 结算接口态：进入通关 / 失败结算前，如服务端仍在处理结果，可短暂显示“结算中”遮罩；若结算失败，保留当前结果页并提示重试提交。

## 功法三选一交互

- 普通层胜利后弹出，BOSS 层胜利不弹出。
- 每次展示 3 张卡，卡片展示品质、名称、门派、流派、局外等级、局内星级和一句效果摘要。
- 点击卡片进入选中态，卡片描边高亮；未选中任何卡片前，“确认选择”按钮默认禁用。
- 点击“确认选择”后加入本局构筑并返回战斗页下一层；提交选择期间按钮需进入短暂 loading / 禁用态，避免重复点击。
- “刷新一次”仅 V0 免费 1 次；使用后按钮置灰并显示“已刷新”。
- 点击“刷新一次”后清空当前选中态，重新生成 3 张卡；刷新结果返回前，卡片与确认按钮保持禁用。
- 已满 3 星的同名功法不再进入随机池；如果因调试强制出现，卡片置灰并标注“已满星”。
- 不可选卡片：满星、无效或异常卡片一律不可进入选中态，点击仅给轻提示，不关闭弹窗。
- 本局已获得 20 本不同名功法后，底部提示“随机池已收束在当前 20 本内”。
- 弹窗关闭：正常战斗流程下不提供直接关闭按钮，必须在 3 选 1 中完成选择；仅调试模式可允许跳过，并需带明显“调试跳过”标识。

## 功法列表与详情状态

- 功法列表需至少覆盖：已解锁、未解锁、可升级、满星、品质筛选 / 标签筛选。
- 未解锁功法：卡片置灰，仅展示品质、门派 / 流派和解锁章节，名称可显示为“未知功法”或半隐藏。
- 可升级功法：展示经验进度条和“可升级”角标，点击详情后主按钮显示“升级”。
- 满星功法：展示“满星”角标，详情页隐藏升级按钮或置灰为“已满星”。
- 品质筛选：V0 至少保留“全部 / N / R / SR”标签；SSR / UR 可在组件规范中保留品质色，但 V0 列表默认不作为主筛选。
- 功法详情浮层：展示品质、名称、门派、流派、基础效果、升星效果、羁绊提示、升级状态；关闭后回到功法列表原滚动位置。
- 关闭回退：关闭详情浮层后，保留列表的当前筛选条件和滚动位置；切出到其他主页面后是否保留原滚动位置，V0 可按页面重进后重置处理。
- 接口态：列表加载中可显示骨架卡；加载失败显示“重试”；无功法数据时显示空提示，不渲染占位假数据。

## 结算回流

- 通关结算：“返回章节”回到章节页并刷新下一章解锁；“继续查看功法”进入功法列表页。
- 失败结算：“再试一次”回到同一章节战斗页并重新开始本局；“返回章节”回到章节页。
- 结算页仅展示本局结果和奖励摘要，不保存前端游戏数据；所有奖励与章节进度以服务端返回为准。

## 调试入口

- 调试入口只在开发 / 原型环境展示，不进入正式 UI 主流程。
- 入口建议位于章节页右上角或战斗页暂停面板内，尺寸低于主按钮，不抢主流程。
- 调试面板展示：当前章节、层、波、角色实时属性、当前功法列表、快速切章、强制胜利、强制失败。
- 调试操作必须带明显“调试”标签，避免与正式按钮混淆。

## 待确认项

- `V0` 正式提测时是否保留“游戏入口页”，还是登录成功后直接进入章节页。
- 功法详情中的“升级”按钮在 V0 是否开放真实升级能力；当前先按“可展示状态、是否可点待确认”处理。
- 调试入口最终固定放在章节页右上角，还是仅放入战斗页暂停面板，需程序结合实现成本确认。
