# V0 章节页状态补充说明

本说明用于补足 `V0_02_章节界面.svg` 未完全展开的章节页状态，便于程序按统一规则实现。

## 适用范围

- 对应页面：`02_界面设计/V0_02_章节界面.svg`
- 对应结构文档：`01_流程与结构/V0_信息架构.md`
- 对应标注文档：`03_切图与标注/V0_界面标注说明.md`

## 页面固定结构

- 顶部：角色信息、当前战力、版本信息、调试入口。
- 中部：章节卡列表区域，默认纵向排布当前章节与后续章节。
- 底部：`章节 / 功法 / 角色 / 商城` 页签栏。

## 默认状态

- 默认进入章节页时，首张卡片为当前可挑战章节。
- 当前章节卡需同时展示：
  - `当前章节` 标签
  - 章节名
  - 层数 / 波数
  - 推荐战力
  - 主按钮 `开始战斗`
- 点击 `开始战斗` 后进入 `V0_03_战斗界面.svg` 定义的三段式战斗页；本轮 `V0` 临时采用文字战斗主表现区，不进入人物 / 敌人图形战斗页。
- 如玩家刚完成上一章通关回流，默认将新解锁章节切为当前章节。

## 已解锁章节

- 展示 `已解锁` 标签。
- 卡片可点击，用于切换当前预览章节。
- 若卡片内保留入口按钮，则按钮文案与当前章节一致，点击后直接进入该章节战斗。
- 已解锁但非当前章节的卡片不显示 `当前章节` 标签。

## 未解锁章节

- 卡片整体降透明度，保留基础章节名和解锁条件。
- 主按钮置灰，不进入战斗页。
- 点击卡片或灰态按钮时，只给轻提示：`通关上一章后解锁`。

## 战力不足提示

- 当玩家当前战力低于推荐战力时，在当前章节卡内显示黄色提示文案：`推荐战力不足，仍可挑战`。
- 提示仅做风险告知，不阻断进入战斗。
- 主按钮仍保持可点击。

## 章节加载中

- 初次进入章节页且接口未返回时，章节卡区域显示 2-3 个骨架卡或统一文案 `章节加载中`。
- 加载中阶段：
  - `开始战斗` 按钮禁用
  - 已开放页签仍可切换
  - 调试入口是否可用由程序按联调需要决定

## 章节请求失败

- 接口失败时，章节卡区域显示失败提示：
  - 标题：`章节加载失败`
  - 说明：`请检查网络或稍后重试`
  - 按钮：`重试`
- 点击 `重试` 后重新请求章节数据。
- 失败态下不展示伪造章节数据。

## 空数据 / 无有效配置

- 当接口返回为空或服务端配置不可用时，章节卡区域显示：
  - 标题：`暂无章节配置`
  - 说明：`请检查服务端章节配置后重试`
- 该状态下：
  - 不进入战斗页
  - 不展示虚拟章节卡
  - 底部已开放页签仍可切换

## 调试入口

- 调试入口只在开发 / 原型环境展示。
- 推荐位置：章节页右上角，小于主操作按钮，不与页面主流程抢视觉焦点。
- 点击后进入调试面板，支持：
  - 快速切章
  - 查看角色实时属性
  - 查看当前功法与羁绊
  - 强制胜利 / 强制失败

## 底部页签

- `章节`：当前页高亮。
- `功法`：可点击进入功法页。
- `角色`、`商城`：V0 仅保留占位，不跳转。
- 点击未开放页签时，统一给轻提示：`暂未开放`。

## 临时约定

- 若程序为降低实现成本，暂时不做骨架屏，可用单行文案替代加载态，但仍需保留加载中、失败、空数据三类区分。
- 若当前版本最终取消“游戏入口页”，章节页仍作为登录成功后的第一个主页面；本说明无需随之重写。
- 章节页进入战斗页时，默认沿用战斗原型中的顶部 HUD、中心文字战报主表现区、底部构筑条三段式骨架，不单独扩展开发调试式战斗面板。
