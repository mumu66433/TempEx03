# V0 信息架构

```mermaid
flowchart TD
  A["账号登录 / 注册层"] --> B["游戏入口页（登录成功后）"]
  B --> C["章节页"]
  C --> D["战斗页"]
  D --> E["普通层通关"]
  E --> F["功法三选一弹窗"]
  F --> D
  D --> G["章节通关结算"]
  G --> C
  D --> H["挑战失败结算"]
  H --> C
  C --> I["功法列表页"]
  I --> J["功法详情浮层"]
```

## 页面职责

- `账号登录 / 注册层`：沿用现有账号弹层，负责输入账号、注册、自动登录和切换账号
- `游戏入口页`：承接账号登录后的第一个正式游戏界面，用于强调“进入游戏”的单一主路径
- `章节页`：展示当前章节、章节名、章节图和开始入口
- `战斗页`：承载层波、角色状态、敌人状态和实时战斗信息
- `功法三选一弹窗`：每层结束后的核心策略节点
- `功法列表页`：查看已解锁功法与局外升级信息
- `功法详情浮层`：查看功法品质、流派、门派和升级收益
- `结算页`：展示挑战结果与下一步操作

## 核心交互路径

1. 玩家在现有账号登录 / 注册层完成登录后进入游戏入口页。
2. 点击开始后进入章节页。
3. 从章节页进入当前可挑战章节。
4. 在战斗页自动推进层波。
5. 普通层胜利后弹出三选一。
6. 章节通关或失败后回到章节页。
7. 玩家可从底部页签进入功法页查看功法详情。

## V0 状态覆盖清单

- `账号登录 / 注册层`：沿用现有实现，不在本目录重复出表单稿。
- `游戏入口页`：默认进入、切换账号、进入游戏。
- `章节页`：默认当前章节、已解锁章节、未解锁章节、推荐战力不足提示、调试入口。
- `战斗页`：普通波、精英波、BOSS 层、受击、死亡、波次切换、暂停。
- `功法三选一弹窗`：默认三选一、卡片选中、刷新一次后置灰、满星功法不可选、本局 20 本收束提示。
- `功法列表页`：全部 / N / R / SR 筛选、已解锁、未解锁、可升级、满星。
- `功法详情浮层`：基础效果、升星效果、羁绊提示、可升级、已满星、关闭回到列表。
- `结算页`：章节通关后解锁下一章、失败后再试一次、返回章节、进入功法列表。

## 接口态覆盖清单

- `章节页`：初次加载中、加载失败可重试、无有效章节配置。
- `战斗页`：战斗初始化中、结算提交中、结算失败提示并允许重试提交。
- `功法列表页`：加载中、加载失败可重试、无可展示功法数据。
- `结算页`：奖励结算中、奖励结算失败提示。

## 临时实现约定

- 暂停面板点击“返回章节”时，先弹二次确认，确认后视为放弃本局。
- 底部 `角色`、`商城` 页签在 V0 仅保留占位，不进入新页面。
- 战斗页底部构筑条默认支持横向滚动，单屏优先展示最近获得或当前排序靠前的功法。
